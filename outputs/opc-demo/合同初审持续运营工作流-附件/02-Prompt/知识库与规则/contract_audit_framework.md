# 企业合同初审规则框架

> 版本：v2.1 · 2026-06-21  
> 适用：企业采购、技术服务、软件实施等商事合同初审  
> 执行：本地规则引擎（确定性）+ 办公小浣熊 @知识库（语义补充）

---

## 一、框架定位

本框架将 **固定检查清单** 与国家级法律法规逐项对应，按 **合同类型** 区分适用边界。

**核心原则：规则引擎是参考层，大模型是决策层。**

| 层级 | 职责 | 载体 |
| --- | --- | --- |
| **规则参考层** | 各审核维度的初筛信号（计算、关键词提示）；供大模型复核，**非终局结论** | `contract_review.py` → `rule_reference_json` |
| **大模型决策层** | 通读合同 + @知识库，逐维给出最终结论，可修正引擎误报 | 办公小浣熊 / CrewAI `contract_auditor` |
| **知识库** | 红线、规则清单、案例、类型矩阵 | `contract_audit_kb.md` 等 |
| **联网尽调** | 主体存续、失信、涉诉 | 小浣熊联网检索 |

规则引擎 **不替代法务终审**；阈值（如违约金日费率万分之五、上限 20%）来自司法实践与内控惯例，已在各条「依据」中标注。

## 四、审核流程

```
合同正文 → 识别合同类型(T1/T2/T3/T4)
         → 按类型筛选适用 R 编号
         → 规则引擎逐条输出（编号/结论/证据/法律依据）
         → @知识库 语义复核（格式条款、隐性冲突、案例）
         → 联网尽调（主体公开风险）
         → 汇总报告 / 周报
```

## 六、版本与维护

- 法规变更 → 更新 `ALL_RULES.legal_basis` 与本文
- 新增合同类型或检查项 → 扩展矩阵 + 实现 `_check_*` + 同步清单文档
- 阈值调整 → 同步法务内控并标注「内控阈值」
