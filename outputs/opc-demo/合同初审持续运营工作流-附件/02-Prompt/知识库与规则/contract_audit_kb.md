## 适用场景

采购合同（T1）、技术服务/软件开发（T2）、承揽安装（T3）、混合合同（T4）的标准化初审。

- **规则参考层** → 本地 `rule_reference_json`：各审核维度初筛信号，供大模型复核，**不是终局结论**

**终局审核报告由大模型结合本文件红线 + 合同全文产出**；可修正规则引擎误报，但须引用原文依据。

## 高风险红线（必须拦截）

> 主要依据：《民法典》470、496、577、585、590条；《仲裁法》5条；《民诉法》35条

1. **争议解决冲突**：同时约定仲裁与诉讼（对应 R23；语义层可进一步审查条款表述）
2. **金额三件套不一致**：大写金额、数字金额、明细合计三者任一不符（对应 R06、R07、R08）。
3. **税务计算错误**：不含税金额 × 税率 ≠ 税额（对应 R08、R09；《增值税法》19—21条）。
4. **付款比例**：分期付款比例合计不等于 100%（对应 R10；《民法典》634条）。
5. **主体不一致**：首部、签章处、正文中的甲乙双方名称不一致（对应 R02）。

## 中风险关注项

- 付款条件仅写「验收后付款」但未定义验收标准（R05；《民法典》621条）。
- 逾期违约金日费率超过 0.05%（R14；《民法典》585条；法释〔2023〕13号65条）。
- 保密义务期限表述为「永久」或缺少例外情形（《民法典》503条等）。
- 知识产权归属约定模糊（背景 IP vs 交付成果；《民法典》845—850条）。

## 法务修改话术模板

| 问题类型 | 建议表述 | 依据 |
| --- | --- | --- |
| 争议解决 | 统一为「提交北京仲裁委员会仲裁，仲裁裁决为终局」 | 《仲裁法》5条 |
| 付款节点 | 明确「签署后 X 日内 / 验收合格后 X 个工作日内」 | 《保障中小企业款项支付条例》9条 |
| 违约金上限 | 增加「累计不超过合同总价的 20%」 | 《民法典》585条（内控惯例） |
| 保密期限 | 改为「合作期间及终止后 3 年，法定保密信息除外」 | 合同编实践 |

## 审核流程（OPC 一人全栈）

1. 识别合同类型（T1/T2/T3/T4）→ 见 `contract_audit_framework.md`
2. 上传合同 → 文档解析抽取字段
3. 规则引擎固定检查（逐条带 R 编号与法律依据，当前见 `contract_review.py`）
4. @知识库 语义复核（本文件红线 + 固定检查清单）
5. 联网尽调乙方主体
6. 数据分析 → 报告 → 周报 PPT
