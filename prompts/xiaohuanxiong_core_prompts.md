# 办公小浣熊核心 Prompt（OPC 高手创造赛 · 可复用资产）

> 对应 **合同初审持续运营工作流** 的定时任务、Agent、联网、知识库、数据分析、PPT 等关键能力，可直接粘贴至办公小浣熊网页端。  
> 工作流说明见 [`docs/OPC_SUBMISSION.md`](../docs/OPC_SUBMISSION.md)。

---

## 模块 0 · 定时任务（每日/每周流水线）

### 0.1 创建每日 09:00 任务

```
请帮我创建一个办公小浣熊定时任务：

【任务名】合同初审-每日拉取
【调度】每个工作日 09:00（请给出 cron 表达式）
【执行内容】
1. 提醒我从 OA 待签目录 / 邮箱附件 / 共享盘拉取「昨日新增待审合同」
2. 对每份合同依次执行：文档解析 → 联网尽调乙方 → @知识库复核 → 生成审核报告
3. 高风险项汇总并提醒法务复核

请输出：cron 表达式、任务描述、失败重试建议（如 09:30 补跑一次）。
```

### 0.2 创建每周五 17:00 任务

> 桌面端可直接粘贴完整正文：`outputs/opc-demo/schedule-weekly-task-prompt.txt`

```
【任务名】OPC演示-每周合同初审周报
【调度】每周五 17:00
【运行入口】云端工作台；开启联网；可 @知识库

每周五 17:00 自动执行 4 步：

1. 【联网检索】巡检本周增值税率、采购/服务合同法规变化；输出变化表 + 来源链接；有变化则生成知识库红线补丁文案
2. 【数据分析】@知识库 读取本周审核记录；汇总通过率、三组规则对比、Top5 高频风险、主体风险分布、效率节约（4h vs 3min）；输出 3 条洞察
3. 【知识库写回】高风险案例 → 合同风险案例库；高风险主体 → 对方主体画像库；法规补丁 → contract_audit_kb
4. 【演示文稿】按 WEEKLY_BRIEFING_PPT.md 生成 6 页采购委员会周报大纲 + PPT 生成 Prompt

规则：数值从 JSON/知识库计算；联网不可用则标「需人工复核」；最后输出高风险清单与下周优先 3 项。
```

## 模块 1 · 文档处理（合同解析）

```
你是企业法务助手。请阅读附件合同全文，抽取并输出 JSON：
- 合同名称、甲乙双方、合同金额（数字+大写）、税率、税额、付款比例、签效期
- 争议解决条款原文
- 保密条款原文
不要编造，缺失字段标为 null。
```

## 模块 2 · 知识库（@知识库 复核 · 读取）

```
我已上传《contract_audit_kb.md》至知识库。请 @知识库 结合高风险红线，
对以下合同争议解决、保密、知识产权条款做语义冲突检测，
输出表格：条款摘要 | 风险等级 | 冲突点 | 推荐修改话术 | 知识库条目编号。
```

### 2.2 知识库 · 写入（沉淀案例）

```
请将本次合同《{file_name}》审核结论摘要写入知识库「合同风险案例库」，格式：
公司名 | 合同类型 | 高风险项（≤3条）| 处理结论 | 日期
不要重复已有条目，若乙方已在「对方主体画像库」请交叉引用。
```

## 模块 3 · 联网检索（对方主体尽调）

```
请开启办公小浣熊联网检索，查询合同乙方「{party_b}」：
1) 工商登记状态（存续/注销/异常）
2) 失信被执行人、经营异常名录
3) 近 12 个月涉诉案件与负面新闻

输出：
- 综合风险等级（高/中/低）
- 关键事实 bullet（每条附来源链接）
- 一段可写入知识库「对方主体画像库」的摘要（≤200字）

若信息不足，明确标注「需人工复核」，不要编造。
```

## 模块 4 · 数据分析

### 4.1 单份合同合规洞察

```
基于以下审核 JSON（固定检查清单结果），生成数据分析结论：
1) 各规则组通过率对比（财务/法务/文本）
2) 高/中/低风险分布
3) 与人工初审（约4小时）的效率对比估算

输出表格 + 3 条管理层可读洞察，不要改写字段数值。

【JSON】
{audit_evidence_json}
```

### 4.2 周报汇总（每周五）

```
以下是本周 {contract_count} 份合同的审核 JSON 数组。请做周报级数据分析：
1) 整体通过率 vs 上周（若未知则只报本周）
2) 三组规则通过率对比
3) Top 5 高频不通过规则（名称 + 次数）
4) 累计节省人工小时数（假设人工 4h/份，AI 3min/份）
5) 3 条下周行动建议

输出 Markdown 表格，数值可从 JSON 计算，不得编造合同不存在的内容。

【JSON 数组】
{weekly_audit_json_array}
```

## 模块 5 · 文案（审核报告）

```
你将审核合同《{file_name}》。系统已完成固定检查清单的结构化校验，证据如下：
{audit_evidence_json}

联网尽调摘要（如有）：
{due_diligence_summary}

请生成中文 Markdown 审核报告：总体结论、关键字段、维度对照表（engine_signal vs 最终结论）、可复制修改建议。

规则参考层 JSON 为初筛信号，**不是终局结论**。数值类优先采信引擎；语义类可修正误报并须引用合同原文；证据不足标需复核。
高风险项优先给出可粘贴进合同的修改话术。
```

## 模块 6 · 汇报/PPT

### 6.1 单份合同 · 6 页汇报大纲

```
请把以下合同审核结论做成 6 页汇报 PPT 大纲（商务风格）：
- 封面：合同名 + 合规通过率
- 背景：人工初审痛点（4小时/份）
- 数据：金额/税率/付款比例
- 总览：固定检查统计
- 风险：Top 5 不通过项
- 决策：修订后复审 / 附条件签署

每页不超过 5 个要点，可直接用办公小浣熊 PPT 生成。
主色建议：深蓝 + 铜色点缀。
```

### 6.2 周报 PPT

```
请根据以下周报数据分析结果，生成 6 页「合同初审周报」PPT 大纲，
结构严格参照 docs/WEEKLY_BRIEFING_PPT.md（封面、概览、合规趋势、Top5 风险、主体尽调、下周行动）。
每页 ≤5 要点，适合粘贴至办公小浣熊 PPT 一键生成。

【周报数据】
{weekly_analytics_markdown}
```

## Agent 编排 · 单份全链路（一次对话）

> **重要**：请在 **云端工作台**（非「本地 Agent」）新建对话。本地 Agent **不提供**联网搜索与云上知识库 API。云端对话中请 @知识库 引用已入库的 `contract_audit_kb.md`、`contract_audit_rules.md`。

```
你是办公小浣熊合同初审 Agent（云端工作台，已开启联网与 @知识库）。对附件合同按顺序执行，每步简短汇报：

1. 【文档处理】抽取关键字段 JSON
2. 【联网检索】尽调乙方工商/失信/涉诉/负面（必须附来源链接）
3. 【知识库 + 规则参考】@contract_audit_kb @contract_audit_rules 复核红线；将本地规则引擎 dimensions 作为必查维度清单，逐维给出 engine_signal vs 最终结论（可修正误报并引用原文）
4. 【数据分析】基于终局结论输出通过率、风险分布
5. 【文案】生成 Markdown 审核报告（含维度对照表）
6. 【知识库写入】主体画像 + 案例摘要
7. 【汇报】6 页 PPT 大纲

规则：不得编造；数值类优先采信规则引擎计算；语义类以你的解读为准并说明理由。
```

### 分步深化 Prompt（逐步跑、便于截图）

| 步骤 | 入口 | Prompt 要点 |
| --- | --- | --- |
| 联网尽调 | 云端工作台 + 联网 | 模块 3，乙方全称 |
| 知识库读取 | @知识库 | 模块 2.1，引用红线库 + 固定检查清单 |
| 知识库写入规则 | @知识库 | 「将 contract_audit_rules 全文确认已入库；若无则生成写入条目」 |
| 案例沉淀 | @知识库 | 模块 2.2 + 尽调摘要 |
| 数据分析 | 数据分析模式 | 模块 4.1，粘贴 `review-response.json` 中 auditJson |
| 周报沉淀 | 数据分析 | 模块 4.2，单份样本时注明「本周 n=1」 |
