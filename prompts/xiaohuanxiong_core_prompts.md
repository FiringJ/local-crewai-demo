# 办公小浣熊核心 Prompt（OPC 高手创造赛 · 可复用资产）

> 以下 Prompt 对应本作品四大能力模块的组合使用，可直接粘贴至办公小浣熊网页端或 SenseNova API 调用。

## 模块 1 · 文档处理（合同解析）

```
你是企业法务助手。请阅读附件合同全文，抽取并输出 JSON：
- 合同名称、甲乙双方、合同金额（数字+大写）、税率、税额、付款比例、签效期
- 争议解决条款原文
- 保密条款原文
不要编造，缺失字段标为 null。
```

## 模块 2 · 知识库（@知识库 复核）

```
我已上传《contract_audit_kb.md》至知识库。请结合知识库中的高风险红线，
对以下合同争议解决、保密、知识产权条款做语义冲突检测，
输出：风险等级、冲突点、推荐修改话术。引用知识库条目编号。
```

## 模块 3 · 数据分析（合规洞察）

```
基于以下审核 JSON（22 条规则结果），生成数据分析结论：
1) 各规则组通过率对比
2) 高/中/低风险分布
3) 与人工初审（约4小时）的效率对比估算
输出表格 + 3 条管理层可读洞察，不要改写字段数值。
```

## 模块 4 · 文案（审核报告）

```
你将审核合同《{file_name}》。系统已完成 22 条规则的结构化校验，证据如下：
{audit_evidence_json}

请生成中文 Markdown 审核报告：总体结论、关键字段、三组规则表格、可复制修改建议。
不得改变结构化结果中的通过/不通过/需复核判断；证据不足须标需复核。
```

## 模块 5 · 汇报/PPT（管理层简报）

```
请把以下合同审核结论做成 6 页汇报 PPT 大纲（商务风格）：
- 封面：合同名 + 合规通过率
- 背景：人工初审痛点
- 数据：金额/税率/付款比例
- 总览：22 项规则统计
- 风险：Top 5 不通过项
- 决策：修订后复审 / 附条件签署

每页不超过 5 个要点，可直接用办公小浣熊 PPT 生成。
```
