# 演示视频分镜脚本（约 3 分 30 秒）

> 主线：一份**真实底稿合同**从写入飞书到主体画像的完整一单。
> 原则：全程真实运行不剪辑造假；每个写回动作都在飞书界面上停留 2-3 秒让评委看清。

## 录制前准备（5 分钟）

```bash
cd ~/Projects/local_crewai_demo

# 1. 让闭环“忘记”NO.003，以便录制时重新审一遍真实底稿合同
uv run python scripts/feishu_contract_loop.py --forget-node QO6XwIY6siV3G3k8eogcs6aLnRb

# 2. 预先打开这些飞书页面（放不同标签页，录屏时切换）：
#    - 知识库「合同审查闭环demo」目录页
#    - 多维表格「合同审查记录」
#    - 【主体画像】上海东旦软件开发有限公司
```

若要演示「写入即审」，也可以先在飞书里把「真实底稿合同-医院绩效系统57.95万」改名/删除重传，效果等价，`--forget-node` 更省事。

## 录制注意

- 终端字体调大（⌘+ 两三次），日志才看得清。
- 运行等待段（约 60-90 秒，rules_agent 模式含 LLM 调用）可以后期 2-4 倍速，但**保留起止时间戳**，呼应「约 3 分钟出报告」。
- 若担心现场网络，提前把跑完的飞书页面也录一遍作备份素材。
- 导出 ≤15MB 提交版可用：`ffmpeg -i in.mov -vcodec libx264 -crf 28 -preset slow -vf scale=1920:-2 out.mp4`
